## 一、设计原则

### 1.1 六维度与 MBTI 映射

CATTI 的 6 维度跟 MBTI 4 轴的对应关系:

| CATTI 代号 | 中文名 | 正向 (+) | 负向 (-) | MBTI 对应轴 |
|---|---|---|---|---|
| **E/I** | 社牛浓度 | 外向 E | 内向 I | **MBTI E/I** |
| **P/L** | 发动机转速 | 主动 P | 松弛 L | ≈ **MBTI N/S**(N=主动探索, S=安于眼前) |
| **T/F** | 冷静抬杠度 | 冷静 T | 感性 F | **MBTI T/F** |
| **J/S** | 强迫症指数 | 有序 J | 随性 S | **MBTI J/P**(J=有序, P=随性) |
| M/D | 恋爱脑指数 | 恋爱脑 M | 清醒 D | 扩展 · 关系亲密度 |
| N/Z | 内耗指数 | emo N | 佛系 Z | 扩展 · 内心压力 |

⚠️ **命名撞车提醒**:
- CATTI `P/L` 里的 `P` = 主动(Proactive),**不是** MBTI 的 P(Perceiving)
- CATTI `J/S` 里的 `S` = 随性(Spontaneous),**不是** MBTI 的 S(Sensing)
- 内部分析时应用 MBTI 原生术语(N/S/J/P)避免混淆

### 1.2 向量 → MBTI 换算规则

从 `vector = [E, P, T, J, M, N]` 推出 MBTI 4 字母:

| MBTI 轴 | 判定 |
|---|---|
| E / I | `vector[0] ≥ 0 → E`, `< 0 → I` |
| N / S | `vector[1] ≥ 0 → N`(直觉主动), `< 0 → S`(感觉稳定) |
| T / F | `vector[2] ≥ 0 → T`, `< 0 → F` |
| J / P | `vector[3] ≥ 0 → J`, `< 0 → P` |

M 和 N(CATTI 扩展维度)不参与 MBTI 判定,是 MBTI 之外的"情感 / 内心"补充。

### 1.3 匹配算法

```
distance(user, cat) = sqrt( sum((user[i] - cat[i])^2) for i = 0..5 )
```

最小距离即本命猫。

## 二、匹配机制诊断 · "中心吸附" 问题

### 2.1 现象

用户在大量问题上选 B(中性, 得分 0)时,用户向量 ≈ `[0,0,0,0,0,0]`。这种情况下匹配器必然选择**向量距离原点最近的猫**,导致总是匹配到均衡型猫(MID / CREAM / TRIC)。Playwright 随机测试也反复验证了这一点。

### 2.3 根因

- **MID**(magnitude 1.41)对任何 `mag < 1.5` 的用户都是"最近猫"
- **CREAM / TRIC** 排名 2、3,mag < 3,吸附力也很强
- **BUG / ORNG** 等极端型 mag > 7,只有用户"很有态度"才能匹配到
- 48% 的用户向量估计落在 mag < 4 的中心区,这片区域由 MID/CREAM/TRIC/STDY/BIG 包圆

### 2.4 解决方案候选

| 方案 | 做法 | 优点 | 代价 |
|---|---|---|---|
| **A · 向量手术** | 把 MID/CREAM 的向量微调,每轴都有 ±1 的偏向 | 最小改动,保留"平衡"人设 | MID 可能失去"真·中庸"味道 |
| **B · 中心惩罚** | 匹配算法加 penalty:用户 mag < 3 时对 mag < 3 的猫 × 1.5 | 算法层解决,不动数据 | 数学不干净,有玄学味 |
| **C · 改题目** | B 选项(中性)改成 ±0.5 分(非 0),逼用户向量离开原点 | 根本解决 | 要重写 18 题选项,工作量大 |
| **D · 加维度** | 新增 N/S(MBTI 直觉/感觉)维度,6D → 7D | MBTI 匹配更严谨 | 每只猫 vector 补第 7 列,要加 3 道新题 |

**推荐**: **A + C** 组合。A 立即见效,C 根本解决。(后续用户拍板)
